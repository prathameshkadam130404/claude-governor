⛽ Governor

A budget-aware agent loop for Claude Code. The model sees its context and its 5-hour / 7-day usage quota every turn — so it economizes under pressure, checkpoints durably, and resumes cleanly instead of dying mid-task.

Install GitHub

The problem

Two limits interrupt long agentic work. Claude models are natively aware of one of them:

Context windowUsage quota (5h / 7d)
Model aware?✅ native❌ never shown to the model
Failure modelossy auto-compactionhard mid-task cutoff

The quota clock is invisible to the model: it will happily spawn three parallel subagents four minutes before your 5-hour window slams shut, and everything they learned dies with the session. Dozens of tools display quota gauges — to the human. Governor puts the gauge in front of the model, every turn, with a policy attached.

What the model sees

[governor] ctx 62% | 5h 87% (resets 14:32, 41m) burn 1.9%/m → dry ~7m
band: WIND-DOWN (5h-quota) — finish the current unit and deliver it to the
user; start NOTHING new; update .governor/RESUME.md with state + next step.

Injected at turn start and after every tool batch mid-turn. Four bands: CRUISE (silent) → ECONOMY (economize) → WIND-DOWN (finish, start nothing) → CHECKPOINT (save state, end cleanly). Burn-rate forecasting races your spend against the reset clock, so 67%-and-hemorrhaging escalates while 87%-and-coasting doesn't.

Install

Two commands in your terminal, one inside Claude Code:

claude plugin marketplace add prathameshkadam130404/claude-governor
claude plugin install governor@claude-governor

Then, inside Claude Code:

/governor:install

Check it worked: send any message, then run /governor:status. Your status bar will show something like:

⛽ CRUISE  ·  ctx 12%  ·  5h ▓▓░░░░░░ 19% ↺4h52m  ·  7d 2% ↺6d

Requires Node.js (Claude Code already runs on it). Quota data needs a Pro/Max subscription; API-key users get context-only mode. Zero dependencies, everything local, no telemetry, MIT.

Example use cases

Example 1 — The 11 PM refactor that doesn't eat itself

You hand Claude a three-part refactor at 87% of your 5-hour window. Without governor, it starts all three, the window slams shut mid-file, and you resume tomorrow to a half-renamed codebase. With governor, the WIND-DOWN directive lands before the first edit: the model declines to start, scopes the work, writes .governor/RESUME.md with the exact order to do it in, and tells you when the window resets. Next session picks the note up automatically and executes it whole.

Example 2 — Research fan-out that survives the cutoff

Four Explore subagents are sweeping a large codebase when the quota dies with no warning. Normally their findings die with the session — resume flags restore the conversation, not the discarded agent work. Governor teed every subagent's final message to .governor/subagents/ the moment each one finished, and machine-wrote RESUME.auto.md from the tool journal at death. The next session starts with "read these 4 preserved outputs before redoing any research" already injected — zero tokens re-spent on what was already learned.

Example 3 — Early warning at 67%, not a panic at 97%

A parallel-agent workload is burning 2%/min. The raw percentage says 67% — comfortably "fine" — but the regression says dry in ~30 minutes, well before the reset. Governor escalates to ECONOMY early: the model switches to targeted reads, batches tool calls, and hands search to cheap subagents. The window stretches to cover the whole task instead of dying at 90% with the finale unwritten. (This exact escalation-below-threshold happened in live validation.)

How it works

1 · Collector

A statusline shim persists the budget JSON Claude Code already emits (context_window, rate_limits) — no scraping, no undocumented APIs.

2 · Injector

Hooks inject the [governor] line with band, burn rate, and projected exhaustion vs. reset — at turn start and mid-turn after every tool batch.

3 · Policy

A skill defines what each band requires, so the directive is enforceable, not vibes.

4 · Durability

Every tool call journaled; every subagent's final message teed to disk; transcript archived before compaction; RESUME.auto.md machine-written on rate-limit death — zero model cooperation needed.

5 · Resume

The next session starts with the checkpoint and preserved subagent outputs injected.

Self-checks

Setup nudge if the collector is missing; a red ⚠ hooks off? statusline marker if the plugin's hooks aren't loaded. Both failure directions announce themselves.

Honest limitations

  • A single uninterrupted text generation has no hook point mid-stream — true for native context awareness too.
  • Quota fields exist only for Pro/Max, after the first API response; everything degrades gracefully to context-only mode.
  • Headless (claude -p) collector cadence is unverified; interactive sessions are the validated path.
  • If Anthropic ships native quota awareness someday: good. The durability layer stays valuable regardless.
